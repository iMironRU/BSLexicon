## Что это

Инструмент для **обучения и тренировки базового синтаксиса** встроенного языка 1С (BSL).
Сознательно реализуется только **язык**, а не платформа: никаких справочников, документов,
регистров, форм, HTTP и криптографии. Зато то, что есть, можно **наблюдать в исполнении** —
пошаговый отладчик показывает стек, переменные и их типы в реальном времени.

Педагогически тренажёр построен на принципе **reading-first / PRIMM**
(Predict → Run → Investigate → Modify → Make): сначала читаем и предсказываем поведение
чужого кода, потом меняем и пишем своё — в одной среде. Подробнее — в [концепции](docs/concept.md).

## Возможности (текущий статус — скелет ядра)

- ✅ **Лексер + парсер BSL** (подмножество MVP) → AST. Двуязычные ключевые слова (`Если`/`If`),
  регистронезависимость, строки с экранированием `""`, комментарии.
- ✅ **Генераторный tree-walking интерпретатор** — исполнение на главном потоке с моделью
  шага «один оператор = один `yield`» (вариант A из концепции, §6).
- ✅ **Живая инспекция переменных** — имя, **текущий тип** (в т.ч. смена типа в рантайме), значение.
- ✅ **Каталог языка** (YAML) как единый источник правды + **инвариант рантайм↔каталог** и
  **doctest** примеров в CI.
- ✅ **Редактор Monaco** + кнопка «Запустить», панели «Вывод» и «Переменные».
- 🚧 Пошаговая отладка в UI (step into/over/out), брейкпоинты, стек вызовов.
- 🚧 Синтакс-помощник: автодополнение/hover/signature help из каталога, панель-справочник.
- 🚧 Коллекции (`Массив`, `Структура`, `Соответствие`, `ТаблицаЗначений`), даты, исключения.

> Подробный скоуп MVP и что отложено — в [концепции, §4](docs/concept.md).

## Быстрый старт

```bash
npm install        # установка зависимостей
npm run dev        # дев-сервер Vite → http://localhost:5173
npm run verify     # типы + инвариант каталога + тесты (то же гоняет CI)
npm run build      # прод-сборка в dist/
```

Отдельные проверки:

```bash
npm run typecheck       # tsc --noEmit по всем проектам
npm run test            # vitest (ядро)
npm run check:catalog   # схема + инвариант рантайм↔каталог + doctest примеров
```

## Архитектура

Принцип из концепции: **чужой парсер — свой интерпретатор**. Самое ценное (исполнитель
с отладкой) пишем сами, потому что пошаговая отладка почти бесплатна лишь тогда, когда
сам управляешь циклом вычисления.

```
                 исходник BSL
                      │
            ┌─────────▼─────────┐
            │   lexer → parser   │   src/core/lexer, src/core/parser
            └─────────┬─────────┘
                      │  AST
            ┌─────────▼─────────────────┐
            │  генераторный интерпретатор │  src/core/interpreter
            │  (yield на каждом операторе)│
            └─────────┬─────────────────┘
                      │  вывод · переменные · события шага
   ┌──────────────────▼──────────────────┐
   │ React-оболочка: Monaco + панели      │  src/app
   └─────────────────────────────────────┘

   Каталог языка (catalog/*.yaml) ── единый источник правды ──▶ рантайм
                                                            ├─▶ Monaco-провайдеры (план)
                                                            └─▶ панель-справочник (план)
```

Каталог и рантайм связаны **инвариантом**: функция в рантайме обязана иметь запись в каталоге
и наоборот — иначе падает сборка (`npm run check:catalog`). Примеры из каталога прогоняются
тем же интерпретатором как doctest, поэтому документация физически не расходится с поведением.

## Структура репозитория

```
catalog/              Каталог языка (единый источник правды)
  schema.json         JSON-схема записи каталога
  functions/*.yaml    Глобальные функции (Сообщить, СокрЛП, ВРег, …)
  types/*.yaml        Типы (Число, Строка, Булево, …)
src/
  core/               Ядро (чистый TS, без фреймворка)
    lexer/            Токены, двуязычные ключевые слова, лексер
    parser/           AST + рекурсивный парсер
    interpreter/      Значения, область видимости, генераторный интерпретатор, встроенные функции
    index.ts          Публичный API ядра: run(), Interpreter, типы
  app/                React-оболочка: редактор + панели
scripts/
  check-catalog.ts    Валидация схемы + инвариант рантайм↔каталог + doctest
tests/                Тесты ядра (vitest)
docs/concept.md       Концепция проекта
.github/workflows/    CI (проверки) и деплой на GitHub Pages
```

## Дорожная карта

| Фаза | Содержание | Статус |
|---|---|---|
| **0** | Каталог языка (YAML) как единый источник правды | 🟢 каркас готов |
| **1** | Лексер/парсер → генераторный интерпретатор с отладкой; Monaco; панель-справочник; runnable-примеры | 🟡 в работе (скелет ядра) |
| **2** | AI-тьютор поверх каталога (NL-поиск, объяснение ошибок) — Pages + тонкий прокси | ⚪ план |
| **3** | Встраиваемый компонент с event-API → обёртка под LMS (LTI/xAPI/SCORM) | ⚪ план |

## Технологии

TypeScript (strict) · React · Vite · Monaco Editor · Vitest · GitHub Actions → GitHub Pages.
Ядро (`src/core`) не зависит от фреймворка и переиспользуемо как встраиваемый компонент.

## Вклад

Новые функции и типы добавляются записью в `catalog/` + реализацией в рантайме — см.
[CONTRIBUTING.md](CONTRIBUTING.md). PR не пройдёт CI, пока каталог и рантайм не согласованы.
