# Как внести вклад

Спасибо за интерес к BSLexicon! Проект открыт для правок через Pull Request.

## Окружение

- Node.js ≥ 20, npm.
- `npm install` — установка зависимостей.
- Перед коммитом гоните `npm run verify` (типы + инвариант каталога + тесты).

## Главный инвариант: рантайм ↔ каталог

Каталог языка (`catalog/`) — **единый источник правды**. Действует жёсткое правило:

> Каждая функция в рантайме (`src/core/interpreter/builtins.ts`) обязана иметь запись
> `kind: function` в каталоге — и наоборот. Расхождение роняет `npm run check:catalog`,
> а значит и CI.

Это не бюрократия, а защита: документация физически не может разойтись с поведением.

## Добавить встроенную функцию (типичный PR)

1. **Рантайм.** Добавьте запись в массив `BUILTINS` в
   [`src/core/interpreter/builtins.ts`](src/core/interpreter/builtins.ts): `id` (русское имя),
   `aliases` (англ. имя в нижнем регистре и пр.), `arity`, `impl`.

2. **Каталог.** Добавьте запись в подходящий файл `catalog/functions/*.yaml`. Поле `id`
   должно **точно совпадать** с `id` из рантайма. Заполните `names` (ru/en), `category`,
   `signature`, `description`, по возможности `params` и `returns`.

3. **Doctest.** Добавьте хотя бы один `examples` с полем `expect` — он будет исполнен тем же
   интерпретатором в CI. Пример:

   ```yaml
   examples:
     - code: Сообщить(ВРег("абв"));
       expect: "АБВ"
   ```

4. **Тест (желательно).** Для нетривиального поведения добавьте кейс в `tests/`.

5. `npm run verify` локально → зелёный → PR.

## Стиль

- TypeScript strict. Ядро (`src/core`) — **без зависимостей от фреймворка**: его можно
  встраивать отдельно (см. дорожную карту, Фаза 3).
- Комментарии и пользовательские строки — по-русски, в тон существующему коду.
- Сообщения коммитов — короткие и по сути.
